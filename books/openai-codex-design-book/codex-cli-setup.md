# Codex CLIを安全に導入する

この章では、Codex CLIを使い始めるための最小構成を扱います。

ただし、単に起動できればよい、という立場は取りません。  
最初から、認証情報、作業ディレクトリ、権限、ログの扱いを意識します。

## まず確認すること

- 作業するリポジトリはGitで管理されているか
- 未コミットの変更は把握しているか
- secretsや`.env`を不用意に読ませない構造になっているか
- Codexに任せる作業範囲が明確か
- 失敗したときに戻せる状態か

## Windows / WSL2 / PowerShell

Windows環境では、PowerShell、WSL2、ネイティブWindowsのどれで動かすかが重要です。

この本では、次の方針で進めます。

- 迷ったらWSL2を基本にする
- Windows固有の検証は別枠で扱う
- パス区切り、改行コード、権限差分を明示する
- CIではローカル環境との差分を残す

## 最初の作業単位

Codexを最初から大きなリファクタリングに使うのは避けます。

最初は、次のような小さな作業が向いています。

- READMEの整備
- テストコマンドの確認
- 既存issueの調査
- 小さなバグ修正
- AGENTS.mdの初期化

## この章の完了条件

この章を読み終えた時点で、読者は次の状態になっていることを目指します。

- Codex CLIを安全な作業ディレクトリで起動できる
- 変更前にGit状態を確認できる
- Codexに渡す作業範囲を小さく切れる
- 認証情報を不用意に読ませない意識を持てる
