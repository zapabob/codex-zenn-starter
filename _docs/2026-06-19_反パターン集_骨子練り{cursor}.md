## B. 10反パターン最終リスト（優先度順）

### #1 秘密コミット
- **症状:** `.env`、`auth.json`、API キー、個人パスが diff に載る。
- **なぜ増えるか:** エージェントが「動かすため」に設定ファイルを丸ごと編集・追加する。人間は diff の行数に圧倒される。
- **検知:** secret scan（gitleaks）、`.gitignore` 漏れ、PR で `sk-` / `Bearer` / `C:\Users\` パターン。
- **処方:** `AGENTS.md` Rules に禁止パス列挙。SOP に「コミット前 `git diff --staged`」。CI に secret scan。
- **関連:** [MILSPEC+`_docs` 記事](https://zenn.dev/zapabob/articles/agents-md-docs-traceability-milspec)、CI ChatGPT 認証 `_docs` ログ。

### #2 曖昧 Done
- **症状:** 「いい感じ」「だいたい動く」でタスク完了扱い。再現手順が無い。
- **なぜ増えるか:** エージェントは自然言語で完了報告する。Commands / Done が repo に無いと検証不能。
- **検知:** Done 節にコマンド名が無い。preview URL や CI green が言及されない。
- **処方:** `AGENTS.md` の Done を「コマンド + 期待出力」で書く。MILSPEC 記事の Done テンプレを参照。
- **関連:** MILSPEC+`_docs` 記事。

### #3 test 省略
- **症状:** lint / test / typecheck を走らせず merge または「完了」報告。
- **なぜ増えるか:** エージェントは変更したファイル周辺だけ見がち。Commands に test が無いと走らない。
- **検知:** PR に test 実行ログなし。CI 初回 red。ローカル `npm test` 未実行の宣言。
- **処方:** `AGENTS.md` Commands に固定。Done に「CI green またはローカル test 通過」。
- **関連:** Codex 設計本 第3章。

### #4 スコープ爆発
- **症状:** 依頼1行に対し、無関係なリファクタ・rename・「ついで」が混ざる。
- **なぜ増えるか:** エージェントは「改善機会」を見つけると止まらない。Rules にスコープ制約が無い。
- **検知:** diff 行数 >> 依頼規模。PR Summary と diff のミスマッチ。
- **処方:** Rules「依頼外変更禁止」「最小 diff」。CI PR review（`codex exec review`）で指摘。
- **関連:** AGENTS-SOP 記事、本 repo CI workflow。

### #5 チャットだけの設計判断
- **症状:** なぜ A 案か、試してダメだった B 案かが、チャット外に残らない。
- **なぜ増えるか:** セッションが切れる。モデルが変わる。チャットは検索しにくい。
- **検知:** 同じ失敗を別セッションで繰り返す。`_docs/` が空。commit message が `fix` のみ。
- **処方:** `_docs/yyyy-mm-dd_{内容}{AI}.md` に「仮説→検証」表。起動時に最新3件を読むルール（グローバル AGENTS.md）。
- **関連:** MILSPEC+`_docs` 記事（中核）。

### #6 AGENTS.md 肥大
- **症状:** 1ファイルに SOP・個人設定・過去経緯・Design が全部。32KiB 接近、読み込みコスト増。
- **なぜ増えるか:** 「エージェントに覚えさせたい」を全部 AGENTS.md に足す。分離の手間を避ける。
- **検知:** 行数 500+。Commands / Rules / Done が埋もれる。言語別手順が AGENTS.md 内に冗長。
- **処方:** 3層（グローバル / repo AGENTS.md / SOP+DESIGN）。symlink またはリンクのみ。
- **関連:** [AGENTS-SOP 記事](https://zenn.dev/zapabob/articles/agents-md-sop-design-symlink)。

### #7 CI とローカル Commands の不一致
- **症状:** ローカルでは `npm test`、CI では別コマンド。または CI だけ secret 前提。
- **なぜ増えるか:** エージェントが CI yaml だけ、または AGENTS.md だけ更新する。
- **検知:** AGENTS.md Commands と `.github/workflows` のコマンドが grep で不一致。
- **処方:** Commands を単一の真実に。CI 変更時は AGENTS.md と `_docs` を同時更新。
- **関連:** Codex CI `_docs` ログ、設計本 第7章。

### #8 MCP / ツール増殖
- **症状:** 使わない MCP が有効、ツール定義が毎ターン載る、勝手に外部サービスを叩く。
- **なぜ増えるか:** 「便利そう」で全部 ON。opt-in 儀式が無い。
- **検知:** MCP 10+ 有効。Hermes/Cursor で意図しない API 呼び出し。コスト・レイテンシ増。
- **処方:** suggest → ユーザー選択 → 実行。repo ごと MCP 許可リスト。AGENTS.md に優先順位。
- **関連:** system prompt 解剖記事（MCP opt-in）、Hermes 比較記事。

### #9 Learned / `_docs` / AGENTS.md の役割混同
- **症状:** セッション学習を AGENTS.md にベタ書き。または `_docs` に読者向け記事を書く。
- **なぜ増えるか:** 「残す」場所の区別が無い。continual learning と repo 憲法が混ざる。
- **検知:** AGENTS.md に「Learned」が無限増殖。`_docs` が Zenn 下書き置き場になる。
- **処方:** AGENTS.md = 規範（短い Learned のみ昇格可）。`_docs` = 飛行記録。`articles/` = 公開物。
- **関連:** MILSPEC 記事、本 repo AGENTS.md Learned 節。

### #10 公開物と内部メモの混在
- **症状:** `articles/` に執筆メモ、内部ニックネーム、価格戦略が入る。
- **なぜ増えるか:** エージェントが「1ファイルにまとめる」を好む。published フラグの意味を無視。
- **検知:** `published: true` な記事に「TODO」「自分用」。読者向けトーンでない段落。
- **処方:** Rules「articles/ は読者向けのみ」。内部は `_docs/` または draft `published: false`。
- **関連:** 本 repo AGENTS.md Learned User Preferences。

## E. AGENTS.md 追記ブロック（コピペ用）

```md
## Anti-patterns (do not)
- Never commit secrets, auth.json, .env, or machine-specific paths.
- Do not mark Done without running Commands (test/lint/preview) and stating results.
- No scope creep: fix only what the request asks; no drive-by refactors.
- Do not store design decisions only in chat — append `_docs/yyyy-mm-dd_{topic}{agent}.md`.
- Keep this file short; move detail to docs/sop/ and docs/design/DESIGN.md.

## Done (required)
- State which Commands ran and whether they passed.
- If CI exists, note expected workflow or local equivalent.
```
